02 · Data Flow — Two Core Journeys
A — Founder Scoring Journey
Onboarding
Account Creation (4 screens)
Founder enters name, email, password, company name, industry, stage, revenue status, problem statement. Supabase auth creates user. Row inserted into founder_profiles.
auth.users founder_profiles
Document Upload
POST /api/profile-builder/upload
PDF/DOCX → pdf-parse extracts text. Claude Haiku runs extraction prompt → returns structured field map + confidence scores per field. File stored in Supabase Storage.
profile_builder_uploads supabase storage Claude Haiku
Smart Q&A (5 sections)
POST /api/profile-builder/extract — per message
User answers targeted questions. Each answer → Haiku extracts fields → returns {extractedFields, confidenceMap, completionScore, followUpQuestion}. Saved per section. Sections: (1) Market Validation (2) Market & Competition (3) IP & Technology (4) Team (5) Financials.
profile_builder_data[section 1–5] founder_profiles.profile_builder_flow
Score Calculation
POST /api/profile-builder/submit
Merges all 5 sections → AssessmentData. Runs calculateQScore() across 30 indicators × 6 parameters. Applies confidence multipliers (stripe=1.0×, document=0.85×, self_reported=0.55×). Sector + stage adaptive weighting. Outputs finalIQ (0–100), grade, iqBreakdown[6], reconciliationFlags, unlockCards.
qscore_history iq_indicator_scores founder_profiles.assessment_completed
Post-Score
Async tasks after submission
AI Actions generated (LLM: "What 5 actions reach 80?") → cached in qscore_history.ai_actions. Feed milestone auto-posted. Score delta computed vs previous_score_id chain. Dashboard realtime update via Supabase channel subscription.
qscore_history.ai_actions (JSONB) feed_posts
B — Agent Chat → Artifact → Score Signal Journey
Message Received
POST /api/agents/chat
Auth validated. conversationId resolved (create new or load existing agent_conversations row). Usage counter checked against subscription_usage table.
agent_conversations subscription_usage
Context Assembly
8-layer system prompt built before any LLM call
Parallel queries: (1) founder_profiles (2) qscore_history latest (3) own agent artifacts [ownArtifacts limit] (4) high-relevance agents' artifacts [always] (5) medium-relevance agents' artifacts [if topic matches] (6) agent_activity events (7) knowledge_library RAG chunks (8) agent_memory session summary. All assembled into one system prompt ≤6000 chars.
agent_artifacts agent_activity agent_memory knowledge_library
LLM Observe Loop
Claude Sonnet — max 5 iterations
LLM responds. If tool_call detected: data tools (Tavily/Apollo) execute → result re-injected → loop continues. If artifact tool called: breaks loop → enters 2-pass artifact generation. If execution tool called: runs action → breaks loop.
Claude Sonnet 4.6 Tavily / Apollo (conditional)
Artifact Generation
2-pass: extract context → generate artifact
Pass 1 (Haiku, extraction): parse conversation history into sparse facts JSON. Pass 2 (Sonnet, generation): build specialized prompt + context → generate structured artifact JSON (3000 tokens, temp 0.55). Quality evaluation (Sonnet, reasoning): score 0–100 completeness, actionability. Self-critique if score < 70 → regenerate with feedback. Saved to agent_artifacts.
agent_artifacts artifact_jobs
Score Signal
applyAgentScoreSignal() — fires after every artifact save
Looks up artifact_type → dimension + boost points from ARTIFACT_BOOST map. Checks qscore_history for source_artifact_type duplicate (prevents double-counting). If new: inserts qscore_history row with data_source='agent_completion'. Also inserts score_evidence row with status='verified'. Dashboard updates in realtime via Supabase channel.
qscore_history score_evidence
Persistence
Messages + memory saved after response
User message + assistant reply inserted to agent_messages. agent_conversations.last_message_at + message_count updated. Session compressed by LLM → agent_memory upserted for next conversation. Cross-agent delegation_tasks created if orchestration enabled.
agent_messages agent_memory delegation_tasks

Safety guardrails, role framing, output format rules. Prepended to every agent.

3

Own Memory
Last N own artifacts (limit = config.memory.ownArtifacts). E.g., Patel loads last 3 ICPs/playbooks. Labelled "What you built together".
6
Cross-Agent Context
highRelevanceAgents: always loaded. mediumRelevanceAgents: loaded if message topic matches domain keywords. Limit = config.memory.otherArtifacts. Labelled "Other advisers built these".
7
Knowledge Library RAG
2 curated domain resources retrieved per message. Semantic search filtered by founder's stage + sector. From knowledge_library table. (Future: per-agent knowledge bases)
8
Conversation Rules + Goals
Agent goal (agent_goals), pending delegations (delegation_tasks), session summary (agent_memory), conversation rules block. Budget enforced: ≤6000 chars total.
Tool Execution Loop — Observe → Act (max 5 iterations)
Tool Type · What fires · Result
Data Tool
web_research, apollo_search, lead_enrich, fetch_stripe_metrics
Executes synchronously. Result re-injected into conversation as [Tool result: X]. LLM loop continues for next iteration. Agent can chain multiple data calls before responding.
Loop continues ↺
Artifact Tool
icp_document, gtm_playbook, financial_summary, …
Breaks loop immediately. Enters 2-pass artifact generation: (1) Haiku extraction pass → sparse context JSON. (2) Sonnet generation pass → full structured artifact. Quality eval + optional self-critique. Saves to agent_artifacts. Fires score signal.

create_deal, send_outreach, initiate_call, schedule_followup
Runs action against external service or DB. Result appended to chat reply. Loop terminates. Logged to agent_activity table for audit trail and cross-agent context.
Loop ends → action logged
05 · IQ Score Calculation Pipeline
Step 1
Input Merge
5 profile_builder_data sections merged → AssessmentData struct. DataSourceMap tags each field: stripe | document | self_reported.
Step 2
30 Indicators Scored
6 parameter scorers run in parallel (P1–P6). Each indicator: rawScore 0–5 in 0.5 steps. Excluded if data missing.
Step 3
Confidence Adjustment
effectiveScore = rawScore × (confidence / 0.90), clamped 0.5–1.0×. Stripe=1.0×, document=0.85×, self_reported=0.55×.
Step 4
Sector + Stage Weighting
Base sector weights × stage multiplier (early/mid/growth). Renormalised to sum=1.0. 8 sectors defined.
Step 5
Final IQ
Σ(effectiveScores) / (activeCount × 5) × 100 − sparsityPenalty. Grade A+→F. Reconciliation flags (vcAlerts) attached, never override score.
// Core formula
finalIQ = ( Σ(rawScore_i × confidenceMultiplier_i) / (activeCount × 5) ) × 100 − sparsityPenalty

// Confidence multiplier
confidenceMultiplier = clamp( confidence / 0.90, min=0.50, max=1.00 )

// Sparsity penalty (max −5 pts)
sparsityPenalty = max(0, (20 − activeIndicatorCount) × 0.5)

// Grade thresholds
A+ ≥ 80 · A ≥ 68 · B+ ≥ 60 · B ≥ 52 · C+ ≥ 44 · C ≥ 36 · D ≥ 26 · F < 26
