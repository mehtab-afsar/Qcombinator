01 · One Agent Turn — Full Orchestration Flow
The complete lifecycle of a single POST /api/agents/chat request. Shows auth, parallel DB reads for context assembly, the Sonnet tool-use loop with three dispatch branches, DB persistence, and the async Haiku summarizer that updates memory for the next turn.

02 · System Prompt Assembly — 8 Layers, All DB Reads
Every data source feeding into the assembled system prompt. Layers 1–2 are hardcoded constants. Layers 3–8 are parallel Supabase reads. A token budget guard trims the memory block if the total exceeds 6,000 chars.

03 · Artifact Generation — 2-Pass Pipeline
Fires when an artifact tool_call is detected in the loop. Pass 1 uses Haiku to extract sparse facts. Pass 2 uses Sonnet to generate the structured artifact JSON. A quality evaluator scores the result and optionally triggers a self-critique regeneration. All post-save operations are async fire-and-forget.

04 · Score Signal Pipeline — applyAgentScoreSignal()
Called after every artifact save. Idempotent per artifact type per user. Applies a quality-adjusted point boost to one of six Q-Score parameters, recomputes the weighted overall score using sector-specific weights, and inserts a new qscore_history row linked via previous_score_id for delta tracking. 
flowchart TD
    classDef db fill:#F5F3FF,stroke:#7C3AED,color:#4C1D95
    classDef noopcls fill:#F3F4F6,stroke:#9CA3AF,color:#6B7280
    classDef okcls fill:#F0FDF4,stroke:#16A34A,color:#14532D

    ENTRY([applyAgentScoreSignal\nsupabase  userId  artifactType  quality\nfeatures/qscore/services/agent-signal.ts]) --> LOOKUP

    LOOKUP[ARTIFACT_BOOST map lookup\nartifactType to dbColumn + label + paramId + points] --> CHK1{boost config\nexists?}
    CHK1 -->|no| NOOP1([return boosted: false\nunknown artifact type])

    CHK1 -->|yes| QUAL[Quality multiplier\nfull = 1.0x\npartial = 0.6x\nminimal = 0.3x\nadjustedPoints = points x multiplier]

    QUAL --> IDEMP[(qscore_history SELECT\nWHERE user_id\nAND source_artifact_type = artifactType)]

    IDEMP --> CHK2{row found?}
    CHK2 -->|yes| NOOP2([return boosted: false\nalready boosted -- idempotent])

    CHK2 -->|no| LATEST[(qscore_history SELECT\nWHERE user_id\nORDER BY calculated_at DESC\nLIMIT 1)]

    LATEST --> CHK3{base score\nexists?}
    CHK3 -->|no| NOOP3([return boosted: false\nno assessment taken yet])

    CHK3 -->|yes| BOOST[Increment parameter score\ne.g. p2_score += adjustedPoints\ncap at 100]

    BOOST --> SECTOR[(founder_profiles SELECT\nindustry to sector)]

    SECTOR --> WEIGHTS[(iq_parameter_weights SELECT\nP1-P6 weights for this sector\nfetchDimensionWeights)]

    WEIGHTS --> RECOMPUTE[Weighted recalculation\nnewOverall = p1 x w1 + p2 x w2 + ... + p6 x w6\ncap at 100]

    RECOMPUTE --> INSERT[(qscore_history INSERT\noverall_score: newOverall\nall pN_scores updated\ndata_source: agent_completion\nsource_artifact_type: artifactType\nprevious_score_id: latest.id)]

    INSERT --> RETURN([return\nboosted: true\npointsAdded  dimensionLabel  newOverall])

    class IDEMP,LATEST,SECTOR,WEIGHTS,INSERT db
    class NOOP1,NOOP2,NOOP3 noopcls
    class RETURN okcls
05 · LLM Routing — lib/llm/router.ts + provider.ts
The router dispatches to Haiku or Sonnet based on task class. The chat route calls the provider directly for streaming. All other routes go through routedText(). The provider wraps the Anthropic SDK with error handling and tool_choice support.

All tables keyed through founder_profiles.user_id with Row-Level Security. qscore_history self-references via previous_score_id for score delta tracking. agent_memory has UNIQUE(user_id, agent_id) — one memory row per agent relationship.
